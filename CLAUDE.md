# このプロジェクトでの行動原則

## 確認なしで進めてよいこと
基本的に許可を求めず実行する。以下のみ事前確認すること：
- データ・ファイルの削除
- 金銭が絡む操作（API課金・決済など）
- 個人情報の外部送信

コードの変更・ファイル編集・スクリプト実行・ライブラリ追加などは確認不要。

## 能動的に考える
ユーザーの指示をそのまま実行するだけでなく、常に目的に照らして最善の方法を考えること。

- 指示された方法より良いアプローチがあれば、実行前に一度提案する
- 実装・変更後に改善余地に気づいたら、その会話内で一度だけ指摘する
- 同じ会話で既に指摘した内容は繰り返さない。次の会話でまとめて提案する
- スコアリング・生成品質・投稿ルールなど、PDCAを回せる観点があれば能動的に提案する

## よく使うコマンド
```bash
python generate_posts.py       # 投稿候補15件生成 → pending_review.txt
python score_posts.py          # 採点・PASSのみ残す
python add_and_sort_queue.py   # キューに追加・ラウンドロビン並び替え
python post_now.py             # キューから1件投稿（スロット判定あり）
```
実行順: generate → score → add_and_sort → post_now

## 重要なgotchas
- X文字数カウント: 日本語等ほぼ全文字=1、サロゲートペア=2、上限280。計測は必ず`sns_utils.x_length()`を使う
- .envのAPIキーが全スクリプトの前提。コードへの直書き禁止
- generate_posts.pyとscore_posts.pyのプロンプト・採点基準は連動。片方変えたら他方も確認する

## 実装後に必ず行うテスト（変更のたびに実施）
スクリプトを新規作成・変更した場合、以下を必ず実行してから完了とする：

1. **環境変数チェック**: `python check_env.py` を実行し、必要なキーが全て存在することを確認
2. **ローカル動作確認**: 変更したスクリプトをローカルで実際に実行してエラーがないことを確認
3. **ログ確認**: 出力ログを読んで期待通りの動作になっているか確認

フットサル監視系のテスト手順:
```bash
python check_env.py                  # 環境変数チェック
python monitor_futsal_web.py         # Web監視（満席なら「空きなし」で正常）
python monitor_futsal_email.py       # メール監視（「対象メールなし」で正常）
```
